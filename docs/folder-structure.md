# Folder Structure

Feature-first under `src/`, with UI / domain / AI / data / validation kept
separate. Future phases (chat, memory, voice, dashboard) add sibling feature
folders without refactoring existing code.

```
src/
  app/
    (auth)/
      login/page.tsx          Magic-link sign-in (redirects signed-in users)
      callback/route.ts       Exchanges auth code/token for a session
    (app)/
      onboarding/page.tsx     Guarded host for the questionnaire
      result/page.tsx         Persona result screen
    page.tsx                  Entry router (→ login / onboarding / result)
    layout.tsx                Root layout + providers
    globals.css               Design tokens (Tailwind v4)

  features/
    auth/
      components/login-form.tsx
    onboarding/
      config/questions.ts     Typed question definitions (the only place questions live)
      components/             welcome / question-field / review / progress / analyzing / onboarding-flow
      schema.ts               Zod answer schema + form helpers (derived from config)
      format.ts               Renders answers for the AI prompt
      actions.ts              submitOnboarding server action (validate → analyze → persist)
    persona/
      queries.ts              Reads active persona + latest assignment
      components/persona-result-card.tsx

  lib/
    supabase/
      server.ts               Server client (RSC / actions / route handlers)
      client.ts               Browser client
      middleware.ts           Session refresh + route protection helper
    ai/
      provider.ts             getAnalysisModel() — the single provider swap point
      analyze-onboarding.ts   generateObject() call
      prompts/persona-assignment.ts
    utils.ts                  cn()

  domain/                     Framework-agnostic business logic
    persona/personas.ts       P1–P5 roster + fallback constant + threshold
    persona/assignment.ts     resolvePersonaAssignment() (PRD F-01 rule)
    emotion/types.ts          EmotionScore

  validation/
    ai.ts                     analysisResultSchema (AI output contract)

  types/
    database.ts               GENERATED by `npm run db:types` (do not edit)

  components/
    ui/                       shadcn primitives
    page-shell.tsx            Centered, calm page container
    theme-provider.tsx

  constants/routes.constants.ts
  providers/query-provider.tsx

middleware.ts                 Root middleware → lib/supabase/middleware.ts

supabase/
  migrations/                 20260604120001_init … _rls … _profile_trigger … _updated_at
  seed.sql                    Idempotent upsert of the 5 personas
  config.toml

docs/                         This documentation
```

## Where things go

- **A new question?** Edit `features/onboarding/config/questions.ts` only — UI, validation, and the prompt all follow.
- **A new persona?** Add a row in `supabase/seed.sql` and an entry in `domain/persona/personas.ts`.
- **Swap the AI provider?** Edit `lib/ai/provider.ts`.
- **A schema change?** Add a migration in `supabase/migrations/`, then `npm run db:reset && npm run db:types`.
